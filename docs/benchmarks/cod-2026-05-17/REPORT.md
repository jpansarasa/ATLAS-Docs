## Methodology

**Goal.** Determine whether the current 7B CoD model (`qwen2.5:7b-instruct` on `ollama-cpu-gen`)
is leaving signal on the floor versus larger or newer alternatives. Signal-bearing tokens —
tickers, company names, sector/industry mentions, and numerics with units — are what
downstream extraction grounds against the instrument catalog → sector → macro score.
If CoD drops them, the downstream pipeline cannot recover.

**Primary metric: token preservation rate.** For each (article, model) pair we ask Claude Opus 4-7
(via Azure Foundry) to identify which ground-truth entities are preserved in the CoD output,
matching flexibly (e.g. `AAPL` ↔ `AAPL`, `Apple` ↔ `Apple Inc.`, `$5.3B` ↔ `5.3 billion`).
Rates are micro-averaged (sum-preserved / sum-total across all articles) so categories with
many entities aren't drowned out by no-entity articles.

**Ground truth.** Generated by Claude Opus 4-7 from raw (trafilatura-cleaned) source HTML/XML,
no manual labels. Same model used for scoring — same definition end-to-end.

**Per-model prompt architecture.**

- **Chunked (≤14B):** mirrors production `cod_initial_summary.txt` + `cod_densification.txt` +
  `cod_coalescing.txt` (PR #342 decontaminated). Chunk threshold 8000 tokens, chunk size 6000,
  5 densification iterations, target 80 words.
- **Single-pass (≥24B):** one prompt requesting a densified summary that explicitly preserves
  tickers, companies, sectors, industries, numerics — same word target. All articles in the
  corpus fit in the 32K context window of these models.

## Deviations from spec

- **Corpus 20, not 75.** CPU inference at ~50–100 s/call across 5 chunked iterations made a 75-article
  corpus infeasible inside the 3–4 h budget. Trimmed corpus is stratified the same way; the
  full 72-article corpus is preserved at `corpus.full72.jsonl` for a future re-run.
- **Model substitutions.** The user-requested `qwen3.5:9b`, `granite4.1:8b`, `gemma4:26b`, and
  `qwen3.6:27b` do not exist in the Ollama library. Substituted closest currently-published
  equivalents: `qwen3:8b`, `granite4:7b-a1b-h`, `gemma3:27b`, `qwen3:30b-a3b-instruct-2507-q4_K_M`.
- **Single-pass variant for ≥24 B.** Per the planning conversation's principle of prompt
  architecture matching model capability — long-context models don't need chunking when the
  article fits in context. All 20 corpus articles fit in 32K.
