# Digest article-grounded narrative ‚Äî design

**Date:** 2026-06-04
**Status:** Design (approved knobs; pending spec review ‚Üí implementation plan)
**Area:** SentinelCollector digest (`DigestNarrativeGenerator` + new selection/text components)

## Problem

The daily/weekly/monthly Sentinel digest narrative is generated by asking the LLM to
summarize the period's **extracted observations**. Post CoD/DSL cutover those
observations are atomic verbatim spans ‚Äî `description` and `text_quote` average
~10 characters ("800 feet", "30%"), with only ~7% being even sentence-fragment
length. There is no rich prose anywhere in the extracted data (by design ‚Äî CoD emits
atomic NUM/CLAIM blocks). So the narrative LLM has nothing substantive to summarize:
the digest is no longer empty (the #607‚Äì#609 fixes restored confidence, sources, and
the vLLM narrative path), but the prose is thin and stitched from fragments.

## Goal

Make the digest narrative read like a real macro-financial summary by feeding the LLM
the **cleaned text of the day's most significant articles**, anchored by the
CoD-extracted figures. Deterministic Tier-1 content (roll-up tables, theme radar,
word cloud) is unchanged ‚Äî only the LLM narrative's **input** changes.

Non-goals: changing extraction, the matrix path, Tier-1 stats, or the digest schema.
Not reviving the per-article density step (`context_summary`) ‚Äî superseded by reading
raw text at digest time.

## Decisions (locked)

- **Text source:** raw article HTML at digest time. Read each selected article's
  `raw_content.raw_file_path` (the `raw-data` dir is mounted read-only into
  sentinel-collector) via `IRawContentService`, clean to text via `ITrafilaturaClient`.
  Work happens once/day for ~25 articles only ‚Äî no extra per-article load.
- **Article selection:** theme-stratified + signal-ranked. Group observations by the
  themes the digest already computes; per theme rank parent articles by extraction
  signal (count √ó confidence of high-confidence obs); take top ~3‚Äì4 per theme; dedup to
  ~25. Guarantees every active theme is covered, led by its meatiest article.
- **Per-article text:** lede-heavy truncation, ~4K chars (~1K tokens) per article.
  News front-loads who/what/numbers, so the lede carries the substance.
- **Fallback:** Tier-1 only. On any failure (missing file, trafilatura, vLLM 400/timeout)
  ship the deterministic roll-ups with no narrative + the existing
  `SentinelDigestNarrativeMissing` alert. No second narrative code path.

## Architecture

Three small, independently testable units:

1. **`DigestArticleSelector`** ‚Äî pure: `(IReadOnlyList<DigestObservationRow>, themes) ‚Üí
   IReadOnlyList<SelectedArticle>`. `SelectedArticle = { RawContentId, Theme, SourceUrl,
   Publisher, IReadOnlyList<ExtractedValue> }` where `ExtractedValue` is the salient
   `{description, value, unit, sourceEntity, atlasSectorCode}` from that article's obs.
   Ranking: group by theme ‚Üí within theme order articles by `Œ£ confidence` of obs with
   `confidence ‚â• floor` ‚Üí take top N per theme ‚Üí dedup by RawContentId (keep highest-
   ranked theme) ‚Üí cap to K (~25). Deterministic, no I/O.

2. **`IDigestArticleTextProvider`** ‚Äî `SelectedArticle ‚Üí string cleanedExcerpt`. Reads
   the raw file via `IRawContentService`, cleans via `ITrafilaturaClient`/
   `ContentNormalizer` (the same components extraction uses), truncates to the per-article
   char cap. Returns empty on read/clean failure (caller drops the article). All I/O and
   failure handling is isolated here.

3. **`DigestNarrativeGenerator.BuildPrompt` (modified)** ‚Äî assembles the prompt from the
   selected articles' `{publisher, theme, cleaned excerpt, extracted values}` plus the
   Tier-1 theme/sector roll-up as aggregate context, and asks the (dedicated vLLM, #609)
   model to synthesize a grounded macro narrative. Unchanged: the catch-block
   Degraded/Unavailable handling and the 80-row cap constant (no longer the narrative's
   primary bound ‚Äî the article budget governs now; keep as a belt-and-suspenders guard or
   retire if unused).

## Data flow

```
DigestService
  ‚Üí query observations + compute themes/stats            (existing)
  ‚Üí DigestArticleSelector(observations, themes)           ‚Üí ~25 SelectedArticle
  ‚Üí for each: IDigestArticleTextProvider.GetExcerptAsync  ‚Üí cleaned text (skip on fail)
  ‚Üí DigestNarrativeGenerator.GenerateAsync(articles+values+aggregates)
  ‚Üí vLLM (dedicated client)                               ‚Üí narrative markdown
  ‚Üí DigestRenderer(Tier-1 tables + narrative)             (Tier-1 unchanged)
```

## Token budget

vLLM (`Qwen2.5-32B-Instruct-AWQ`) `max_model_len = 32768`.
- Reserve ~3K instructions + values, ~3K output ‚Üí ~26K for article text.
- Per-article cap ‚âà 26K / 25 ‚âà 1K tokens (~4K chars).
- **Dynamic:** compute `perArticleCharCap = floor(textBudgetTokens / selectedCount) √ó
  charsPerToken`, so fewer themes/articles ‚Üí more text each. Hard-cap total assembled
  prompt below a safety threshold (e.g. 30K tokens) and drop the lowest-ranked articles
  if exceeded ‚Äî never rely on the server to reject (that was the #608 bug class).

## Error handling

| Failure | Behavior |
|---|---|
| `raw_file_path` missing/unreadable | Skip that article, `LogWarning`, continue |
| trafilatura/clean fails for an article | Skip (or stripped-raw fallback), continue |
| 0 articles selected (empty window) | Skip narrative ‚Üí Tier-1 only (as today) |
| vLLM HTTP 400 | `Degraded` + `LogError` (existing #608 path) |
| vLLM timeout/outage | `Unavailable` (existing) ‚Äî alert fires |

## Observability

- Reuse `SentinelDigestNarrativeMissing` (fires on Degraded/Unavailable).
- Add a counter for articles selected vs. successfully read (so silent file-read decay is
  visible): e.g. `sentinel_digest_narrative_articles{outcome=selected|read|skipped}`.

## Testing

- `DigestArticleSelector`: theme-stratified ranking; per-theme top-N; dedup keeps the
  higher-ranked theme; K cap; empty input ‚Üí empty.
- Token budget: per-article cap enforced; total assembled prompt stays under the safety
  threshold; dynamic cap widens when few articles.
- Text provider: missing file ‚Üí empty (article dropped); trafilatura failure handled.
- Integration: narrative generates on vLLM from article text, `llmStatus=Ok`, output is
  substantively longer/richer than the fragment baseline.

## Out of scope / follow-ups

- Reviving the CoD density step for other consumers.
- Weekly/monthly windows feed far more than 25 articles' worth of themes ‚Äî the same
  theme-stratified selection applies, but tuning K per period type is a follow-up.
- Relevance ranking beyond extraction-signal (e.g. publisher trust weighting) is a
  follow-up if coverage quality needs it.
